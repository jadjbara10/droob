===== الخطوة 2: إصلاح مشكلة الترميز في الداشبورد =====

2.1 افتح الملف D:\trans_app\droob\dashboard\src\lib\auth.ts (أو أي ملف فيه API client)
وتحقق من الآتي:
- إذا وجدت res.text() ثم JSON.parse: غيّرها إلى res.json()
- أضف في headers: 'Accept': 'application/json; charset=utf-8'

2.2 افتح كل ملف .tsx في dashboard/src/ وشغّل على كل واحد:
powershell -Command "$content = [System.IO.File]::ReadAllText('PATH', [System.Text.UTF8Encoding]::new($false)); [System.IO.File]::WriteAllText('PATH', $content, [System.Text.UTF8Encoding]::new($false))"
هذا يضمن أن كل ملف محفوظ بـ UTF-8 without BOM.

2.3 في ملف next.config.js أو next.config.mjs أضف:
experimental: { optimizePackageImports: ['lucide-react'] }

2.4 شغّل:
cd D:\trans_app\droob\dashboard
chcp 65001
npx tsc --noEmit
npm run build

2.5 إذا فشل البناء: توقف وأخبرني بالخطأ. لا تعدل شيئاً.

2.6 إذا نجح:

لا تعمل git push
